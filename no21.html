No.21 基礎

確率

目標解答時間：15分

1 問題

箱の中に0から9までの数字が1つずつ書かれたカードが1枚ずつ合計10枚入っている．この箱の中から同時に3枚のカードを無作為に取り出し，取り出されたカードに書かれた3つの数字を大きいものから順に百の位，十の位，一の位として3桁の整数を作る．その整数を $N$ とする．

  1. $N$が10の倍数となる確率を求めよ．
  2. $N$が2の倍数となる確率を求めよ．
  3. $N$が3の倍数となる確率を求めよ．

【条件の整理】

  • 事象の総数： 10枚から3枚を「同時に」取り出すため、組合せ ${}_{10}\mathrm{C}_3$ 通りが分母となる。
  • 整数の作られ方： 取り出した3枚の数字を「大きい順に並べる」というルールがある。つまり、3枚の組合せが1つ決まれば、作られる整数 $N$ はただ1つに決定する。（例：{2, 7, 5} が出たら必ず 752 になる）。よって、並び替え（順列）を考慮する必要はない。
  • 百の位が0になることはないか？： 大きい順に並べるため、3枚のうち最大の数が百の位になる。もし最大の数が0であれば 000 となるが、カードは各1枚しかなく「異なる3つの数字」を取り出すため、最大の数が0になることは絶対にない（最小の組合せでも {2, 1, 0} → 210）。常に必ず3桁の整数になる。

2 解説

問題を解くための考え方から、答案、検算、発展までを順に確認します。

解法選択のトリガー

「同時に取り出す」＋「自動的に並び順が決まる」と来たら

順列（$\mathrm{P}$）は使わず、純粋に組合せ（$\mathrm{C}$）だけで考えます。「取り出したカードのセット」と「作られる整数」が1対1に対応しているため、条件を満たす「セットの個数」を数えればよいのです。

倍数の判定条件

10の倍数： 一の位が 0
2の倍数： 一の位が 偶数 (0, 2, 4, 6, 8)
3の倍数： 各位の数の和が 3の倍数

「大きい順に並べる」というルールと倍数条件の結びつき

$N$ の「一の位」は、取り出した3つの数字の中で「一番小さい数」になります。したがって、「10の倍数である」ことは「取り出した3つの数字のうち、最小のものが 0 である」ことと同値です。

思考プロセス

(1) 10の倍数になる条件
一の位が 0 になる、つまり「取り出した3枚のうち最小の数が 0 である」ことが条件です。
これはシンプルに「0のカードを含んでいればよい」ということになります。（0より小さい数はないため、0が含まれていれば必ず一の位に配置されます。）
したがって、「0を必ず選び、残り2枚を1〜9から選ぶ」組合せの数を求めます。
(2) 2の倍数になる条件
一の位（＝3枚のうち最小の数）が偶数（0, 2, 4, 6, 8）になる場合を考えます。
最小の数が指定されるので、「最小の数が $k$ であるような組合せ」を場合分けして足し合わせます。
例えば、最小の数が 2 の場合、残り2枚は「2より大きい数（3〜9）」から選ばなければなりません。
(3) 3の倍数になる条件
各位の和が3の倍数になる条件です。これは、0から9までの数字を「3で割った余り」でグループ分けして考えるのが定石です。
・余り0のグループ：{0, 3, 6, 9}
・余り1のグループ：{1, 4, 7}
・余り2のグループ：{2, 5, 8}
3つの数の和が3の倍数になるのは、「同じグループから3つ選ぶ」または「各グループから1つずつ選ぶ」のどちらかしかありません。

実際の答案

10枚のカードから同時に3枚を取り出す方法の総数は

\[ {}_{10}\mathrm{C}_3 = \frac{10 \times 9 \times 8}{3 \times 2 \times 1} = 120 \text{ （通り）} \]

これらは同様に確からしい。

（1）

取り出した3つの数字を $a, b, c \ (a > b > c)$ とすると、$N$ は $100a + 10b + c$ と表される。
$N$ が10の倍数となるのは、一の位 $c$ が 0 のときである。
0から9までの数のうち、0が最小であるから、$c=0$ となる条件は「取り出した3枚の中に0のカードが含まれていること」と同値である。
0のカード1枚を必ず選び、残り2枚を1から9の9枚から選べばよいので、その選び方は

\[ {}_9\mathrm{C}_2 = \frac{9 \times 8}{2 \times 1} = 36 \text{ （通り）} \]

したがって、求める確率は

\[ \frac{36}{120} = \frac{3}{10} \]
答: $\dfrac{3}{10}$

（2）

$N$ が2の倍数となるのは、一の位 $c$（取り出した3枚のうち最小の数）が偶数（0, 2, 4, 6, 8）のときである。最小の数 $c$ ごとに場合分けをする。

  • $c=0$ のとき： (1)より $36$ 通り。
  • $c=2$ のとき： 残り2枚は3から9の7枚から選ぶので、${}_7\mathrm{C}_2 = \dfrac{7 \times 6}{2 \times 1} = 21$ 通り。
  • $c=4$ のとき： 残り2枚は5から9の5枚から選ぶので、${}_5\mathrm{C}_2 = \dfrac{5 \times 4}{2 \times 1} = 10$ 通り。
  • $c=6$ のとき： 残り2枚は7から9の3枚から選ぶので、${}_3\mathrm{C}_2 = 3$ 通り。
  • $c=8$ のとき： 残りは9しかなく、2枚選べないため $0$ 通り。

これらは互いに排反であるから、$N$ が2の倍数となる場合の数は

\[ 36 + 21 + 10 + 3 = 70 \text{ （通り）} \]

したがって、求める確率は

\[ \frac{70}{120} = \frac{7}{12} \]
答: $\dfrac{7}{12}$

（3）

$N$ が3の倍数となるのは、取り出した3つの数字の和が3の倍数になるときである。
0から9までの10個の数字を、3で割った余りによって3つのグループに分類する。
$A_0 = \{0, 3, 6, 9\}$ （4個）
$A_1 = \{1, 4, 7\}$ （3個）
$A_2 = \{2, 5, 8\}$ （3個）
3つの数字の和が3の倍数になるのは、次の [I] または [II] の場合である。

  • [I] 同じグループから3つの数字を選ぶ場合
    ・$A_0$ から3つ選ぶ：${}_4\mathrm{C}_3 = 4$ 通り。
    ・$A_1$ から3つ選ぶ：${}_3\mathrm{C}_3 = 1$ 通り。
    ・$A_2$ から3つ選ぶ：${}_3\mathrm{C}_3 = 1$ 通り。
    計 $4 + 1 + 1 = 6$ 通り。
  • [II] 3つのグループからそれぞれ1つずつ選ぶ場合
    ・$A_0$ から1つ、$A_1$ から1つ、$A_2$ から1つ選ぶので、
    ${}_4\mathrm{C}_1 \times {}_3\mathrm{C}_1 \times {}_3\mathrm{C}_1 = 4 \times 3 \times 3 = 36$ 通り。

[I] と [II] は互いに排反であるから、$N$ が3の倍数となる場合の数は

\[ 6 + 36 = 42 \text{ （通り）} \]

したがって、求める確率は

\[ \frac{42}{120} = \frac{7}{20} \]
答: $\dfrac{7}{20}$

場合分けの整理（(2) 2の倍数）

(2)で「最小の数」ごとに場合分けをしました。なぜそのような計算になるのか、視覚的に整理してみましょう。
最小の数（一の位）が決まると、百の位と十の位になれる数は「それより大きい数」に限定されます。

最終結果と検算

  • (1) $\dfrac{3}{10}$
  • (2) $\dfrac{7}{12}$
  • (3) $\dfrac{7}{20}$
検算ポイント：(2) 奇数になる確率による補集合の確認
(2)で偶数（2の倍数）になる確率を求めましたが、補集合である「奇数になる確率」を求めて和が1になるか確認してみましょう。
奇数になるのは、一の位（最小の数）が奇数 (1, 3, 5, 7, 9) のときです。
・$c=1$ のとき：${}_8\mathrm{C}_2 = 28$ 通り
・$c=3$ のとき：${}_6\mathrm{C}_2 = 15$ 通り
・$c=5$ のとき：${}_4\mathrm{C}_2 = 6$ 通り
・$c=7$ のとき：${}_2\mathrm{C}_2 = 1$ 通り
・$c=9$ のとき：作れない（0通り）
合計 $28+15+6+1 = 50$ 通り。
偶数の70通りと合わせると $70 + 50 = 120$ 通りとなり、全体の場合の数 ${}_{10}\mathrm{C}_3$ と完全に一致しました。これで安心して次に進めます。

発展・一般化：「大きい順」の強力さ

本問の最大のポイントは、「取り出した3つの数字を大きい順に並べる」というルールです。もしこのルールがなく「無作為に並べて3桁の整数を作る」であった場合、問題は途端に複雑になります。
なぜなら、「0を選んだ場合、0が百の位になってしまうと2桁の整数になってしまうため、0を含む場合と含まない場合で並べ方の場合分けが必要になる」からです。
本問では、「大きい順に並べる」というルールがあるおかげで、0が必ず一の位に押し出され、「百の位が0になる例外」を一切考える必要がなくなり、単純な「組合せ」の計算だけで処理できるようになっています。条件の意図を正しく読み取ることが、解法をシンプルにする鍵です。

受験生のつまずきポイント

順列（$\mathrm{P}$）を使って計算を複雑化してしまう

「3桁の整数を作る」という言葉につられて、「10枚から3枚選んで並べるから分母は ${}_{10}\mathrm{P}_3$ だ！」としてしまうミスが多発します。「大きい順に並べる」という指定があるため、{2, 5, 8} という組み合わせが出た時点で 852 以外の並び方は発生しません。並び方の自由度がない場合は「組合せ（$\mathrm{C}$）」で考えなければなりません。

3の倍数の条件を「各グループから1つずつ」だけにしてしまう

(3)で「和が3の倍数 $\Rightarrow$ 余り0, 1, 2のグループから1つずつ選ぶ」ことには気づけても、「同じグループから3つ選ぶ（余り0+0+0, 余り1+1+1, 余り2+2+2）」というパターンを完全に抜け落としてしまう誤答が非常に多いです。3の倍数のグループ分けを使う問題では、この「同じグループから3つ」が常に隠れていることを意識しましょう。

アクティブリコール（復習テスト）

今日の学習内容を思い出すことで、記憶を定着させましょう。質問をクリックして答えを確認してください。

Q1. 「取り出した3つの数字を大きい順に並べて整数を作る」という問題において、順列（$\mathrm{P}$）ではなく組合せ（$\mathrm{C}$）を用いて計算してよい理由は何ですか？
選ばれた3つの数字のセットに対して、それらを大きい順に並べる方法は「ただ1通り」に自動的に決まるためです。並べ方の自由度がないため、単に「3つの数字を選ぶ組合せ」だけを数えれば、作られる整数の個数と一致します。
Q2. 取り出した3つの数字を「大きい順に」並べたとき、その3桁の整数が偶数（2の倍数）になるかどうかの判定は、どこに注目すればよいですか？
一の位、すなわち「取り出した3つの数字のうち、最も小さい数」が偶数であるかどうかに注目します。そのため、計算時は「一番小さい数」を固定して場合分けを行うとスムーズに数え上げられます。
Q3. 0から9までの数字から3つを選んで「和が3の倍数」になるようにしたいとき、数字をどのようなグループに分けて考えますか？また、選ぶパターンの大分類は？
数字を「3で割った余り」によって、余り0、余り1、余り2の3つのグループに分けます。和が3の倍数になるのは、「同じグループから3つの数字を選ぶ」パターンか、「3つのグループからそれぞれ1つずつ選ぶ」パターンのどちらかです。
